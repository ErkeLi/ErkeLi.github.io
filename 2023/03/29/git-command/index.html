Git的基本操作

2023年03月29日 留言

讲了git的常用命令，并且后续会继续更新。

常用操作

移动打开到D盘,,ceshi
cd d:

显示当前目录
pwd

进入编辑模式后，连续按两次大写字母Z退出编译模式
ZZ

显示文件所作的改变
git diff readme.txt

查看文件类容
cat readme.txt

配置信息

将git和远程的github仓库相连接
git remote add origin https://github.com/ErkeLi/testgit.git

配置仓库姓名和邮箱，最好一开始就配置
git config –global user.name “ErkeLi”
git config –global user.email “TYWen19@qq.com“

文件上传与下载

将文件添加到缓存区
git add a.txt

将文件上传，注意要加-m，不然会进入编译区
git commit -m a.txt

回到上一个版本
git reset –hard HEAD^

当修改的内容还没有 add 和 commit时，可以用下面的操作来取消修改
git restore readme.txt

当文件被添加的时候（add），可以用下面命令来退出
git restore –staged readme.txt

将工作去所作的所有修改删除，有点类似 退回上一个版本和
git restore readme.txt

创建仓库分支

1、创建分支 main， main可以自定义名称
git branch -M main

2、将本地数据上传到分支 Main，第一次推送master分支时，加上了 –u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，
git push -u origin main

二次上传操作，不用加-u
git push origin main

克隆代码到本地

从git上复制项目到本地
git clone https://github.com/chaooo/hexo-theme-BlueLake.git

从GitHub上下载代码，仓库的代码放到本地themes/BlueLake下，上面的方式会吧代码放到hexo-theme-BlueLake文件夹下
$ git clone https://github.com/chaooo/hexo-theme-BlueLake.git themes/BlueLake
